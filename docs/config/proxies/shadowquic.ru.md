# ShadowQUIC

```{.yaml linenums="1"}
proxies:
- name: shadowquic
  server: www.example.com
  port: 10443
  type: shadowquic
  username: username
  password: password
  # ip: 127.0.0.1
  # quic-versions: [v1]
  # udp-over-stream: false
  # zero-rtt: false
  # keep-alive-interval: 10000
  # congestion-controller: cubic
  # up: 100 Mbps
  # down: 100 Mbps
  # ignore-client-bandwidth: false
  # cwnd: 32
  # bbr-profile: "standard"
  # max-datagram-frame-size: 1400
  # max-open-streams: 1024
  # recv-window-conn: 0
  # recv-window: 0
  # disable-mtu-discovery: false

```

[Общие поля](./index.md)

[Поля TLS](./tls.md)

## username

**Обязательно**, имя пользователя для аутентификации ShadowQUIC.

## password

**Обязательно**, пароль для аутентификации ShadowQUIC.

## ip

**Необязательно**, используется для переопределения результатов DNS-запроса адреса сервера, указанного в опции «server».

## quic-versions

**Необязательно**, устанавливает поддерживаемые версии QUIC. Доступные значения: `v1`/`v2`. По умолчанию `v1`.

## udp-over-stream

**Необязательно**, устанавливает, следует ли передавать UDP через поток (UDP over stream). По умолчанию `false`.

## zero-rtt

**Необязательно**, устанавливает, следует ли включить рукопожатие QUIC 0-RTT (Zero Round Trip Time).

!!! "warning"
    Путь 0-RTT на официальном сервере может считывать состояние пользователя до завершения аутентификации JLS (подтверждено влияние на версию v0.3.11); при использовании официального сервера необходимо отключить zero-rtt на стороне сервера.

## keep-alive-interval

**Необязательно**, интервал времени между отправками пакетов проверки активности (heartbeat) для поддержания соединения, в миллисекундах.

## congestion-controller

**Необязательно**, устанавливает алгоритм управления перегрузкой. Доступные значения: `cubic`/`new_reno`/`bbr`. По умолчанию `cubic`.

## up

**Необязательно**, задает скорость отдачи (загрузки в сеть) клиента, поддерживаются единицы `Mbps`. Заполнение этого поля включает алгоритм управления перегрузкой Brutal; требуется использование mihomo ShadowQUIC на обоих концах соединения.

## down

**Необязательно**, задает запрашиваемую клиентом скорость скачивания, поддерживаются единицы `Mbps`, ограничивается скоростью отдачи принимающей стороны. Заполнение этого поля включает алгоритм управления перегрузкой Brutal; требуется использование mihomo ShadowQUIC на обоих концах соединения.

## ignore-client-bandwidth

**Необязательно**, используется для алгоритма Brutal. Указывает, следует ли игнорировать ограничение исходящей скорости скачивания (`down`) клиента и использовать вместо него ограничение входящего слушателя (listener `down`) или автоматическую настройку. По умолчанию `false`.

## cwnd

**Необязательно**, устанавливает начальный размер окна перегрузки (Initial Congestion Window size). По умолчанию `32`.

## bbr-profile

**Необязательно**, устанавливает степень агрессивности алгоритма BBR. Доступные значения: `standard`/`conservative`/`aggressive`. Работает только в том случае, если для `congestion-controller` установлено значение `bbr`.

## max-datagram-frame-size

**Необязательно**, устанавливает максимальный допустимый размер кадра датаграммы в байтах. По умолчанию `1400`.

## max-open-streams

**Необязательно**, устанавливает максимальное количество одновременно открытых параллельных потоков. По умолчанию `1024`.

## recv-window-conn

**Необязательно**, устанавливает размер окна приема на уровне соединения. Если установлено значение `0`, используется системное значение по умолчанию.

## recv-window

**Необязательно**, устанавливает размер окна приема на уровне отдельного потока. Если установлено значение `0`, используется системное значение по умолчанию.

## disable-mtu-discovery

**Необязательно**, устанавливает, следует ли отключить обнаружение пути MTU (Path MTU Discovery). Если установлено значение `true`, будет использоваться фиксированный размер MTU.
